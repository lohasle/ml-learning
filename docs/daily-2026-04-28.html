2026年4月28日 - 第20课：Agent — 让 LLM 自主行动

今日学习主题

理解 AI Agent 的核心架构：LLM + 工具 + 规划 + 记忆，从被动问答到自主行动的关键飞跃。

为什么学这个？

上一课（RAG）让模型能"查资料再回答"，但模型依然是被动的——你问什么它查什么。Agent 是质的飞跃：模型自己决定要不要查、查什么、用什么工具、需要几步、何时停止。

从架构视角看，RAG 是"给模型装了一个外接硬盘"，Agent 是"给模型装了一双手和一个日程表"。

在 AI 演进中的位置：Agent（2023-2025 爆发）是大模型从"工具"进化为"系统"的关键节点。它让 LLM 从被调用的 API 变成能自主编排工作流的执行者。这也是 Transformer → LLM 阶段的收尾课——至此，你已掌握从线性回归到智能体的完整链条。

核心概念解释

1. Agent 的四大组件

  • 直觉：Agent = 大脑（LLM）+ 手（工具）+ 计划表（规划）+ 笔记本（记忆）
  • LLM：负责理解意图、推理决策、生成回答
  • Tools：搜索引擎、计算器、数据库、API、代码执行器——一切外部能力
  • Planning：任务分解、推理策略（ReAct、CoT、ToT）
  • Memory：短期记忆（对话上下文）+ 长期记忆（向量数据库）
  • 代码对应：Notebook 中的 ToolRegistry（工具）+ ReActAgent（规划循环）+ history（记忆）

2. ReAct 推理范式

  • 直觉：像人解决问题一样——"想想现在该干嘛" → "做一下" → "看看结果" → "再想想"
  • 循环：Thought（推理）→ Action（行动）→ Observation（观察）→ Thought → ... → Final Answer
  • 类比：你在做一道数学题，先想思路（Thought），算一步（Action），看看对不对（Observation），不对就换思路
  • 关键论文：ReAct: Synergizing Reasoning and Acting in Language Models（Yao et al., 2022）

3. Function Calling / Tool Use

  • 直觉：LLM 不是真的"调用"函数，而是输出一段 JSON，告诉框架"我想用这个工具，参数是这个"
  • 流程：用户提问 → LLM 输出 {"action": "search", "input": "..."} → 框架执行搜索 → 结果回传 LLM → LLM 继续
  • 为什么重要：这是 Agent 的"神经肌肉接头"——LLM 只负责"想"，工具负责"做"
  • 生产实践：OpenAI Function Calling、Anthropic Tool Use、Gemini Function Calling

4. Agent 的记忆系统

  • 短期记忆：当前对话的上下文窗口（8K-128K tokens）
  • 长期记忆：向量数据库存储历史交互、用户偏好、知识积累
  • 工作记忆：当前任务的中间结果、scratchpad
  • 类比：短期 = 你正在想的事情，长期 = 你的笔记本，工作 = 草稿纸

关键公式直觉

概念 方法/公式 直觉解释
ReAct 循环 Thought → Action → Observation → 循环 像人一样"想一步做一步"
Tool Selection argmax P(tool_i | query, history) LLM 根据上下文选最合适的工具
Prompt 模板 [Instructions] + [Tools] + [History] + [Query] 把所有信息打包给 LLM 做决策
停止条件 Action = "finish" 或 step ≥ max 防止 Agent 无限循环
Function Calling JSON: {name, arguments} → 执行 → result LLM 输出结构化指令，框架执行

动手实践要点

  • Notebook 实现了完整的 ReAct 循环：ReActAgent 类
  • ToolRegistry 演示了工具注册和调用的标准模式
  • MockLLM 模拟了 LLM 的决策过程（真实场景用 OpenAI/Claude API）
  • 注意 Agent 的最大步数限制——真实系统必须防止无限循环

常见问题

Q: Agent 和普通 LLM 对话有什么区别？ A: 普通对话是一次性的——你问它答。Agent 是循环的——它自己决定要调什么工具、看什么结果、要不要继续。核心区别是自主性和多步推理。一个简单的判断标准：如果系统有"工具选择 → 执行 → 再推理"的循环，它就是 Agent。

Q: 为什么 Agent 经常"跑偏"或死循环？ A: 这是 Agent 工程最大的挑战。原因包括：LLM 选错工具、工具返回无关信息导致推理偏移、没有好的停止条件。解决方案：设置最大步数、添加反思步骤（让 Agent 评估自己刚才的行动是否有效）、用更好的 Prompt 模板。

Q: 现在应该用哪个 Agent 框架？ A: 学习阶段建议先从零写（理解原理），然后 LangGraph 做复杂工作流，OpenAI Assistants API 做快速生产。避免一上来就用最重的框架——先搞懂 Agent 循环的本质，再选工具。

本课总结

  1. Agent = LLM + Tools + Planning + Memory，核心是从"被动回答"到"自主行动"
  2. ReAct（推理+行动）是最经典的 Agent 推理范式
  3. Function Calling 是 LLM 调用外部工具的标准接口
  4. Agent 的核心挑战：可靠性、可控性、评估
  5. 框架选型要看场景，理解原理比选框架更重要
