随机森林与集成学习

🎯 核心概念

1. 基础定义

  • 类型：集成学习方法（可用于分类和回归）
  • 适用场景：几乎所有结构化数据的分类/回归任务，尤其适合快速基线模型
  • 一句话定义：组合多个决策树（或其他弱学习器），通过投票或平均得到更稳定、更准确的预测

2. 集成学习两大流派

  • Bagging（并行训练）：随机森林

- 每个学习器独立训练，看到不同的数据子集

- 最终通过投票/平均降低方差

  • Boosting（串行训练）：AdaBoost、GBDT、XGBoost

- 每个学习器重点修正前一个学习器的错误

- 通过迭代降低偏差

3. 随机森林的双重随机性

  • 数据随机：Bootstrap 有放回抽样，每棵树看到约 63% 的不同样本
  • 特征随机：每次分裂只从 $\sqrt{d}$ 个随机特征中选择（$d$ 为总特征数）
  • 两种随机性共同确保树之间的差异性最大化

4. 数学原理

  • 单棵树误差较大、方差高
  • $N$ 棵独立树投票，方差降为原来的约 $1/N$
  • 前提：树之间的相关性越低，效果越好

📊 关键公式

公式含义直觉
Bootstrap 抽样有放回地从 $n$ 个样本中抽 $n$ 个每棵树看到 ~63% 的独特样本
投票：$\hat{y} = \text{mode}(h_1, ..., h_N)$多数投票少数服从多数
特征子集：$m = \sqrt{d}$每次分裂考虑的特征数避免所有树选中同一强特征
OOB 误差未被抽中的 ~37% 样本做验证免费的验证集

🔗 关联概念

  • 前置知识：决策树（第 5 课）、过拟合与正则化（第 4 课）
  • 后续应用：XGBoost/LightGBM（现代 Boosting）、深度学习中的 Dropout
  • 同类对比：Bagging vs Boosting vs Stacking

💡 记忆技巧

随机森林 = 一群决策树各看不同的数据 + 各看不同的特征 + 最后投票

就像一个委员会：每个人拿到不同的资料，从不同角度分析，最后举手表决。
