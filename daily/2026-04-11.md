## 今日学习主题
**从 if-else 到智能分裂：决策树如何用树状思维做分类**

## 为什么今天学这个？

前四课学完了线性模型（回归 + 分类）和评估/正则化。线性模型假设特征与目标之间是线性关系，但现实中很多决策是"分层判断"的——比如医生诊断：先看体温，再看病程，再看化验指标。决策树正是模拟这种人类直觉判断的算法，它也是最直观、最易解释的 ML 算法，同时也是后续随机森林、梯度提升树等集成方法的基础。

## 这个知识点在 AI 演进中的位置

- **1960s**：概念学习系统（CLS）、最早的树形决策模型思想萌芽
- **1984**：Breiman 等人提出 **CART**（Classification and Regression Trees），奠定现代决策树基础
- **1986**：Quinlan 提出 **ID3** 算法，引入信息增益作为分裂准则
- **1993**：Quinlan 改进为 **C4.5**，处理连续特征和缺失值
- **2001**：Breiman 提出 **随机森林**，决策树作为基础构建块
- **2014-至今**：XGBoost、LightGBM、CatBoost 等梯度提升树在 Kaggle 和工业界大放异彩

> 决策树不是"过时的算法"，而是现代集成学习方法的地基。

## 核心概念解释

### 1. 什么是决策树？

一棵倒过来的树：
- **根节点**：包含所有训练数据
- **内部节点**：对一个特征做判断（如"年龄 > 30？"）
- **叶子节点**：最终预测结果（类别或数值）

```
            年龄 > 40？
           /          \
        是              否
       /                  \
  收入 > 5万？         学历？
   /      \           /      \
 是        否       本科+     本科-
贷！     不贷      贷！      不贷
```

### 2. 树怎么长出来的？（核心问题）

关键思路：**每一步选最好的问题来分裂数据**。

过程：
1. 从根节点开始，包含所有数据
2. 遍历所有特征、所有可能的分裂点
3. 选择让子节点"最纯"的分裂（即分类最明确的）
4. 对子节点递归执行，直到满足停止条件

### 3. 怎么衡量"纯度"？

三个经典指标：

| 指标 | 思路 | 特点 |
|------|------|------|
| **信息增益 (Info Gain)** | 分裂后熵减少了多少 | ID3 使用，偏好多值特征 |
| **信息增益率** | 信息增益 / 固有信息 | C4.5 使用，修正多值偏好 |
| **基尼不纯度 (Gini)** | 随机抽取两个样本类别不同的概率 | CART 使用，计算更快 |

## 直觉理解 / 类比理解

### 20个问题游戏
玩"20个问题"猜一个东西：你不会一开始就问"它是猫吗？"，而是问"它是活的吗？"这样能把可能性砍掉一大半。决策树就是在做这件事——**每一步都选信息量最大的问题**。

### 人事筛选
HR 筛简历不会同时看所有条件，而是先卡学历，再看经验，再看薪资期望——层层过滤。决策树的分裂过程完全一样。

## 关键机制或关键公式

### 信息熵（Entropy）

衡量数据集的"混乱程度"：

```
H(S) = -Σ p_i × log₂(p_i)
```

- 全是同一类 → 熵 = 0（完全确定）
- 各类均匀分布 → 熵最大（最混乱）

### 基尼不纯度（Gini Impurity）

```
Gini(S) = 1 - Σ p_i²
```

- 全是同一类 → Gini = 0
- 计算比熵快（不需要 log 运算），sklearn 默认使用

### 分裂选择

```
选择特征和分裂点，使得：
  父节点不纯度 - 加权子节点不纯度  最大
```

## 最小示例

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris
from sklearn.tree import DecisionTreeClassifier, plot_tree
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, classification_report

# 加载经典鸢尾花数据集
iris = load_iris()
X, y = iris.data, iris.target

# 只取前两个特征，方便可视化
X_2d = X[:, :2]
X_train, X_test, y_train, y_test = train_test_split(
    X_2d, y, test_size=0.3, random_state=42
)

# 训练决策树
tree = DecisionTreeClassifier(max_depth=3, random_state=42)
tree.fit(X_train, y_train)

# 评估
y_pred = tree.predict(X_test)
print(f"准确率: {accuracy_score(y_test, y_pred):.2%}")
print(classification_report(y_test, y_pred, target_names=iris.target_names))

# 可视化决策树结构
plt.figure(figsize=(14, 8))
plot_tree(tree, feature_names=iris.feature_names[:2],
          class_names=iris.target_names, filled=True, rounded=True)
plt.title("决策树结构（鸢尾花数据集，深度=3）")
plt.tight_layout()
plt.savefig('decision_tree_iris.png', dpi=150)
plt.show()

# 可视化决策边界
def plot_decision_boundary(model, X, y, title):
    h = 0.02
    x_min, x_max = X[:, 0].min() - 0.5, X[:, 0].max() + 0.5
    y_min, y_max = X[:, 1].min() - 0.5, X[:, 1].max() + 0.5
    xx, yy = np.meshgrid(np.arange(x_min, x_max, h),
                         np.arange(y_min, y_max, h))
    Z = model.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)
    
    plt.figure(figsize=(10, 6))
    plt.contourf(xx, yy, Z, alpha=0.3)
    plt.scatter(X[:, 0], X[:, 1], c=y, edgecolors='k', s=40)
    plt.xlabel(iris.feature_names[0])
    plt.ylabel(iris.feature_names[1])
    plt.title(title)
    plt.tight_layout()
    plt.show()

# 对比不同深度
for depth in [1, 2, 3, None]:
    t = DecisionTreeClassifier(max_depth=depth, random_state=42)
    t.fit(X_train, y_train)
    acc = accuracy_score(y_test, t.predict(X_test))
    label = f"深度={'无限制' if depth is None else depth}"
    plot_decision_boundary(t, X_2d, y, f"{label} | 准确率: {acc:.2%}")
```

### 关键超参数速查

```python
DecisionTreeClassifier(
    max_depth=5,          # 最大深度（最常用！控制过拟合）
    min_samples_split=10, # 分裂所需最小样本数
    min_samples_leaf=5,   # 叶子节点最小样本数
    max_features='sqrt',  # 分裂时考虑的特征数
    criterion='gini',     # 'gini' 或 'entropy'
)
```

## 关键项目、关键论文或关键系统案例

### 工业应用
- 信用评分：规则透明，监管友好
- 医疗诊断辅助：决策路径可解释
- 故障检测：if-else 规则直接部署到嵌入式系统

## 今天学完后我应该记住什么

1. **决策树本质**：递归地选最优特征和分裂点，把数据分成越来越纯的子集
2. **纯度度量**：信息熵、基尼不纯度，衡量分得好不好
3. **可解释性**：决策树是"白箱模型"，每个预测都能解释
4. **过拟合风险**：不加限制的树会一直长到每个叶子只有1个样本，必须剪枝或限制深度
5. **核心超参数**：`max_depth` 是控制树复杂度的第一旋钮
6. **重要地位**：决策树是随机森林和梯度提升树的基石，值得打牢基础
