# 2026年4月24日 - 第16课：Transformer 架构

## 今日学习主题
**理解 Transformer 如何用 Self-Attention + 位置编码完全替代 RNN，成为现代 AI 的统一架构。**

## 为什么今天学这个？
上一课学了 Attention 机制——解决了 Seq2Seq 的信息瓶颈。但模型底层仍然是 RNN，存在顺序计算、长距离衰减等问题。

2017 年「Attention Is All You Need」论文提出 Transformer：**完全抛弃 RNN，只用 Attention**。这是 AI 演进史上最关键的架构变革之一——今天所有大模型（GPT、BERT、Claude、Gemini）都是 Transformer 的变体。

## 在 AI 演进中的位置
```
RNN/LSTM → Attention机制 → ★ Transformer ★ → BERT/GPT → 大模型时代
```
Transformer 是连接「序列模型时代」和「大模型时代」的桥梁。

## 核心概念

### 1. 多头注意力（Multi-Head Attention）
- **单头的问题：** 只能学一种「看哪里」的模式
- **多头：** 同时从多个角度关注输入，每个头学不同的注意力模式
- **类比：** 多个人同时读文章，分别关注人名、时间、地点、因果关系

| 参数 | 原始论文值 | 含义 |
|------|-----------|------|
| d_model | 512 | 模型维度 |
| h (头数) | 8 | 注意力头数量 |
| d_k = d_v | 64 | 每个头的维度 = 512/8 |

### 2. 位置编码（Positional Encoding）
- Self-Attention 是「集合操作」，不知道 token 顺序
- 用正弦/余弦函数给每个位置生成唯一向量
- 低频维度捕获「大致位置」，高频维度捕获「精确位置」

| 公式 | 说明 |
|------|------|
| PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) | 偶数维度 |
| PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model)) | 奇数维度 |

### 3. Encoder Block 完整结构
```
Multi-Head Attention → Add & LayerNorm → FFN → Add & LayerNorm
```

### 4. 三种 Transformer 变体

| 变体 | 代表 | 用途 |
|------|------|------|
| Encoder-Only | BERT | 文本理解、分类 |
| Decoder-Only | GPT 系列 | 文本生成 |
| Encoder-Decoder | T5 | 翻译、序列转换 |

## 关键设计直觉

| 设计 | 为什么这样选 |
|------|-------------|
| 残差连接（Add） | 防止深层网络梯度消失，让信息有「直达通道」 |
| LayerNorm（不是 BatchNorm） | 与序列长度无关，小 batch 也能稳定训练 |
| FFN 两层 MLP | 增加非线性表达能力（Attention 是线性加权） |
| 正弦位置编码 | 无需学习，可泛化到训练时没见过的序列长度 |

## 常见问题

**Q: Transformer 比 RNN 快多少？**
A: 训练时快很多，因为所有位置可以并行计算。但推理时生成 token 仍然需要逐步进行（自回归）。序列长度 n 的 attention 复杂度是 O(n²)。

**Q: 为什么用 LayerNorm 而不是 BatchNorm？**
A: BatchNorm 依赖 batch 内其他样本的统计量，batch size 小时不稳定。LayerNorm 对单个样本独立归一化，不受 batch size 影响。

**Q: GPT 为什么只用 Decoder？**
A: 生成任务只需要「看前面的 token 预测下一个」，所以用 Masked Self-Attention（遮住后面的位置）。Encoder 的双向注意力对生成任务反而「作弊」了。

## 本课总结
1. Transformer = Self-Attention + 位置编码 + 残差连接 + LayerNorm
2. 多头注意力让模型从多个角度同时关注输入
3. 位置编码解决了 Self-Attention 无序的问题
4. 三种变体（Encoder-Only / Decoder-Only / Encoder-Decoder）对应不同任务
