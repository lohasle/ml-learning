# 2026年4月23日 - 第15课：Attention 机制

## 今日学习主题
**理解 Attention 机制如何让模型「回头看」序列中的关键信息，突破 RNN 的信息瓶颈。**

## 为什么学这个？
上一课（RNN/LSTM）我们看到序列模型通过隐藏状态传递信息，但 encoder-decoder 架构中，encoder 必须把所有信息压缩到一个固定长度的向量——这就是信息瓶颈。

Attention 是 AI 演进史上的**关键转折点**：
- 它最初是为了解决翻译模型的信息瓶颈而提出的（2014 Bahdanau）
- 它后来演变成 Self-Attention，直接催生了 Transformer（2017）
- 今天所有大模型（GPT、BERT、Claude、Gemini）的底层都是 Attention

**这节课是从 RNN 时代跨入 Transformer 时代的桥梁。**

## 核心概念解释

### 1. 信息瓶颈问题
- **直觉：** 你要翻译一句 50 个词的话，但只能用一张便利贴记笔记——信息必然丢失
- **RNN 的问题：** encoder 最后一个隐藏状态要承载所有输入信息，序列越长越困难
- **Attention 的解法：** 让 decoder 每一步都能「回头看」所有输入，自动分配注意力

### 2. Q / K / V 机制
- **Query (Q)：** 「我现在需要什么信息？」——decoder 当前状态
- **Key (K)：** 「每个输入位置有什么信息？」——encoder 各位置的标签
- **Value (V)：** 「每个输入位置的实际内容」——encoder 各位置的内容
- **图书馆类比：** Q=搜索词，K=书名，V=书的内容

### 3. Scaled Dot-Product Attention
- Q 和 K 做点积 → 得到相关性分数
- 除以 √d_k → 防止高维时 softmax 退化为 one-hot
- Softmax → 归一化为概率分布（注意力权重）
- 权重 × V → 加权求和得到输出

## 关键公式直觉

| 概念 | 公式 | 直觉解释 |
|------|------|----------|
| 相关性分数 | score = Q · K^T | Query 和每个 Key 的相似度 |
| 缩放 | score / √d_k | 高维时点积方差大，需要压回来 |
| 注意力权重 | α = softmax(scores) | 每个 Key 位置的「重要程度」(0~1, 总和=1) |
| 输出 | output = α · V | 按重要性加权取出 Value |

## 动手实践要点
- Notebook 中手动实现了 `scaled_dot_product_attention`，不依赖任何框架
- 注意观察注意力权重矩阵：每行是一个 query 对所有 key 的注意力分布
- **缩放因子的效果：** d_k=512 时，不缩放的 softmax 几乎变成 one-hot，缩放后分布更均匀
- 常见坑：忘记缩放 → 训练初期梯度消失

## 常见问题

**Q: Attention 和 Self-Attention 有什么区别？**
A: Attention 通常指 Cross-Attention（Q 来自 decoder，K/V 来自 encoder）。Self-Attention 中 Q/K/V 都来自同一个序列——每个位置都看其他所有位置。Self-Attention 是 Transformer 的核心。

**Q: 为什么除以 √d_k 而不是 d_k？**
A: 假设 Q 和 K 的各分量独立、均值为 0、方差为 1，则点积的方差为 d_k。除以 √d_k 让方差回到 1，保持 softmax 输入的数值范围稳定。除以 d_k 就过度缩放了。

**Q: Attention 的计算复杂度是多少？**
A: O(n²)，n 是序列长度。这是 Attention 的主要瓶颈，也是后续很多优化工作（FlashAttention、稀疏 Attention 等）的动机。

## 本课总结
1. Attention 解决了 Seq2Seq 的信息瓶颈：decoder 不再依赖单一上下文向量
2. Q/K/V 机制让模型自动学习「看哪里、看多少」
3. 缩放因子 √d_k 对训练稳定性至关重要
4. 从 Attention 到 Transformer 只差一步：把 Attention 用在同一个序列上（Self-Attention）
