# 2026年4月26日 - 第18课：大模型微调（Fine-tuning）

## 今日学习主题
**理解大模型微调的核心方法——从全量微调到 LoRA，掌握如何让预训练模型高效适配你的任务。**

## 为什么学这个？
上一课我们学了 BERT 和 GPT 这样的预训练语言模型。它们是「通才」，但实际应用需要「专才」。微调就是从通才到专才的关键桥梁。这是现代 AI 工程师最常面对的实际工作——不是从零训练模型，而是微调已有模型。

## 核心概念解释

### 1. 微调的基本思路
- **直觉**：就像大学毕业生入职后的岗前培训。基础能力已经具备，只需要针对性地学习岗位技能。
- **原理**：保持预训练学到的权重作为起点，用任务特定数据继续训练，调整参数以适配目标任务。
- **代码直觉**：Notebook 中对比了三种方式（Prompt / LoRA / Full FT）的参数量和效果。

### 2. Full Fine-tuning
- **直觉**：把整个模型的所有参数都更新。就像重新装修整栋房子。
- **问题**：7B 参数模型需要 ~28GB 显存来微调，70B 需要 ~280GB。成本极高。
- **适用场景**：数据充足、任务差异大、追求极致效果。

### 3. LoRA（Low-Rank Adaptation）
- **直觉**：不拆房子，只在墙上贴装饰板。用极少的参数（< 0.1%）来近似权重更新。
- **核心公式**：`ΔW = A × B`，其中 A 是 d×r，B 是 r×d，r 远小于 d。
- **为什么有效**：预训练模型已经学到了很好的特征表示，微调只需要在低秩子空间中做小幅调整。
- **代码直觉**：Notebook 演示了 d=100 时，原参数 10,000 → LoRA 只需 800（压缩 12.5 倍）。

### 4. QLoRA 与其他 PEFT 方法
- **QLoRA** = LoRA + 4-bit 量化，进一步降低显存，单张 24GB 显卡可微调 7B 模型。
- **其他 PEFT**：Adapter（插入小网络层）、Prefix Tuning（学习虚拟 prompt）、Prompt Tuning。

## 关键公式直觉

| 概念 | 公式/方法 | 直觉解释 |
|------|----------|----------|
| 微调目标 | min L(f_θ, D_task) | 在任务数据上最小化损失，θ 从预训练值开始 |
| LoRA 更新 | W' = W + A×B | 原权重不变，加一个低秩修正 |
| LoRA 缩放 | ΔW = (α/r) × A×B | α 控制更新幅度，r 越小压缩越大 |
| 秩选择 | r ∈ {4, 8, 16, 64} | 常用值，越大表达力越强但参数越多 |
| QLoRA | NF4 量化 + LoRA | 把原权重压缩到 4bit，只训练 LoRA 参数 |

## 动手实践要点
- Notebook 中演示了 LoRA 低秩近似的可视化效果
- 观察参数压缩比：维度越大，LoRA 的优势越明显
- 训练曲线对比：LoRA 接近 Full FT 的收敛效果，但显存需求大幅降低
- 关键超参：秩 r、缩放因子 α、学习率

## 常见问题

**Q: LoRA 的 r 设多大合适？**
A: 经验上 r=8 是很好的起点。简单任务 r=4 就够，复杂任务可以 r=16~64。r 越大不总是越好——过大可能过拟合，且失去了参数效率的优势。

**Q: 微调需要多少数据？**
A: 取决于方法。Full FT 通常需要数千到数万条；LoRA 可以更少，几百条到数千条就能看到效果；QLoRA 在极少量数据（< 1000 条）上也能工作。

**Q: 微调后如何部署？**
A: LoRA 训练完后可以把 A×B 合并回原权重 W' = W + A×B，推理时零额外开销。也可以不合并，用多个小 LoRA 适配器对应不同任务，动态切换。

## 本课总结
1. 微调 = 让预训练的「通才」变成你任务的「专才」
2. Full Fine-tuning 改全部参数，效果好但成本高
3. LoRA 只调 0.1% 参数就能接近全量效果，是工业界主流方案
4. QLoRA + 量化让单卡微调大模型成为现实
5. 实际工程中 LoRA 是首选，Full FT 是追求极致时的备选

## 下一步
下一课我们将学习 **RAG（检索增强生成）**——另一种让大模型适配特定领域的方法，不需要改模型参数，而是通过外部知识库来增强模型输出。这与微调形成互补：微调改模型内部的「知识」，RAG 给模型外部的「参考书」。
