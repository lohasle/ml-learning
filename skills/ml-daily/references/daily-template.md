# Daily MD 结构模板

每个 daily 文件必须包含以下结构：

## 结构规范

```markdown
# YYYY年M月D日 - 第N课：课程名

## 今日学习主题
**一句话概括今日学习的核心内容**

## 为什么学这个？
说明这个知识点在学习路线中的位置和意义，以及与前一课的关系。

## 核心概念解释

### 1. 概念名
- 直觉解释（不用数学先说人话）
- 公式（如有）
- 代码直觉（对应 Notebook 中的哪段代码）

### 2. 概念名
（同上）

### 3. 概念名
（同上）

## 关键公式直觉

| 概念 | 公式 | 直觉解释 |
|------|------|----------|
| ... | ... | ... |
| ... | ... | ... |

## 动手实践要点
- Notebook 中的关键步骤说明
- 参数选择的理由
- 常见坑点提示

## 常见问题

**Q: 问题1？**
A: 回答

**Q: 问题2？**
A: 回答

## 本课总结
1. 要点1
2. 要点2
3. 要点3

## 下一步
下一课预告和与前课的衔接说明
```

## 质量底线

- 文件大小 > 1000 bytes
- 包含"关键公式直觉"表格
- 包含至少 2 个 Q&A
- 必须与对应 notebook 内容一致（知识点不矛盾）
