# RAG — 检索增强生成

## 🎯 核心概念

### 1. 基础定义
- **类型**：知识增强技术 / 架构模式
- **适用场景**：大模型需要访问外部知识、减少幻觉、回答实时性问题
- **一句话定义**：在 LLM 生成回答前，先从外部知识库检索相关文档，拼接到 prompt 中作为参考

### 2. 数学原理
- 核心公式：cos(q, d) = q·d / (‖q‖×‖d‖)，计算 query 与文档的语义相似度
- 公式直觉：query 和文档的"语义GPS坐标"方向越一致，越相关

### 3. 实践要点
- Chunk size 通常 256-1024 tokens，top-k 通常 3-5
- Embedding 模型选择比向量数据库更重要
- Reranker 可以显著提升检索精度（但增加延迟）

## 📊 关键公式

| 公式 | 含义 | 直觉 |
|------|------|------|
| cos(θ) = A·B/(‖A‖‖B‖) | 余弦相似度 | 方向越一致越相关 |
| TF-IDF = tf × log(N/df) | 词频-逆文档频率 | 在这篇文章常见但在全局少见的词重要 |
| Score = Encoder(q, d) | Cross-Encoder 重排 | 精确判断一对 query-doc 的相关性 |

## 🔗 关联概念
- 前置知识：Embedding、Transformer、大模型微调
- 后续应用：Agent（工具调用）、多模态 RAG
- 同类对比：微调（改模型能力）、Long Context（扩大窗口）
